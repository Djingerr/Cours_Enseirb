
## Introduction

Les réseaux informatiques reposent sur un ensemble de **normes**, **standards** et **organisations internationales** permettant d’assurer :

- l’interopérabilité entre systèmes,
- la compatibilité matérielle et logicielle,
- la stabilité des technologies,
- la sécurité et la fiabilité des communications.

Sans normalisation, chaque constructeur proposerait ses propres protocoles, rendant impossible un Internet mondial cohérent.

---

# 5.1 Pourquoi normaliser ?

La normalisation permet :

- d’éviter les incompatibilités,
- d’unifier les méthodes de communication,
- d’assurer la pérennité des technologies,
- de permettre à divers constructeurs de coopérer,
- de réduire les coûts (économies d’échelle).

Elle est essentielle pour les réseaux publics (Internet) où des milliards d’équipements doivent communiquer ensemble.

---

# 5.2 Les grandes organisations de normalisation

Voici les principales institutions qui définissent les standards réseau.

---

## ISO — International Organization for Standardization

- Organisation internationale reconnue mondialement.  
- A créé le **modèle OSI**.  
- Normalise des protocoles, des formats, des méthodes.

**Exemples :**
- ISO 7498 — modèle OSI  
- ISO 27000 — normes de sécurité  

---

## IEEE — Institute of Electrical and Electronics Engineers

Organisation majeure dans les réseaux et télécommunications.

Elle normalise notamment :

- **Ethernet (IEEE 802.3)**  
- **WiFi (IEEE 802.11)**  
- Bluetooth (IEEE 802.15)  

Les normes IEEE 802 sont fondamentales pour tous les réseaux locaux.

---

## IETF — Internet Engineering Task Force

Organisation ouverte qui élabore les standards d’Internet au travers des **RFC** (Request For Comments).

- TCP  
- UDP  
- IP  
- HTTP  
- DNS  
- TLS  
- SMTP  
- BGP  

Chaque standard Internet existe sous forme de RFC.

---

## ITU — International Telecommunication Union

Agence de l’ONU dédiée aux télécommunications mondiales.

Normalise :

- les réseaux mobiles (3G, 4G, 5G),  
- les codecs audio/vidéo,  
- les liaisons longues distances.

---

## W3C — World Wide Web Consortium

Organisme pilotant les technologies du Web :

- HTML  
- CSS  
- JavaScript  
- Web APIs  

Il assure la cohérence du Web moderne.

---

# 5.3 Les RFC : fondement d’Internet

Une **RFC** est un document technique décrivant un protocole ou une norme Internet.

### Caractéristiques
- ouverte publiquement  
- numérotée  
- évolutive  
- révisée par la communauté  

### Exemples importants
- RFC 791 : protocole IP  
- RFC 793 : TCP  
- RFC 768 : UDP  
- RFC 2616 / RFC 7230 : HTTP  
- RFC 1034/1035 : DNS  

Chaque protocole fondamental possède une RFC qui fait foi.

---

# 5.4 Standardisation matérielle : IEEE 802

Le groupe **IEEE 802** définit les normes LAN / MAN :

| Norme | Fonction |
|--------|----------|
| **802.3** | Ethernet (filaire) |
| **802.11** | WiFi (sans-fil) |
| **802.1D / 802.1Q** | Commutation, VLAN |
| **802.15** | WPAN (Bluetooth) |

Ces normes garantissent l’interopérabilité des équipements réseau.

---

# 5.5 Processus de normalisation Internet (IETF)

Le processus suit plusieurs étapes :

1. **Internet-Draft (I-D)**  
2. Discussions, tests, contributions  
3. **RFC proposée**  
4. Validation  
5. RFC adoptée comme standard

Les standards évoluent selon les besoins technologiques.

---

# 5.6 Pourquoi plusieurs organisations ?

Chaque organisme couvre un domaine différent :

- **IEEE** → technologies physiques et locales  
- **IETF** → protocoles Internet  
- **ISO** → modèles et référentiels généraux  
- **ITU** → télécommunications mondiales  
- **W3C** → technologies Web  

Elles sont complémentaires et assurent la cohérence globale.

---

# Synthèse

La normalisation :

- garantit la compatibilité entre équipements,
- assure la stabilité d’Internet,
- permet l’évolutivité des technologies,
- repose sur des organisations majeures comme l’ISO, IEEE, IETF, ITU et W3C,
- s’appuie sur les RFC pour les protocoles Internet.

Comprendre les organismes et leurs rôles permet de mieux appréhender la structure globale du monde des réseaux.
